Background

This document describes how to configure Inex camera(s) to work with IZCloud.

Applicable Product

A IZCloud-integrated Inex camera running with the latest RoadView release.

You Will Need

  • Computer with Windows 10 or above - with .NET 4.5 enabled in "Windows Features"

  • Chrome or Edge browser

Purchasing an IZCloud Subscription

Contact Inex to purchase an IZCloud subscription. Inex will provide further instructions on how to configure the IZCloud Integration in RoadView

Configuring IZCloud in RoadView

  • Open a browser with the the IP address of the RoadView computer address you configured (usually running on the camera).
    For example:
    192.168.5.110

  • Enter the default username/password = root/root

  • You should see the RoadView Live (Journal) tab.

  • Click on the Settings tab at the top of the screen

  • Scroll down to the IZCloud Integration section

  • Enable IZCloud Integration and set the parameters according to instructions from Inex.

    Remember to click the Save button at the end of the Detector Configuration section after making changes.
Figure 1. Settings Tab - IZCloud Integration Section

The IZCloud server has different Vehicle Lists containing registered plate reads. These lists can be entered in IZCloud manually, or via integration with a 3rd party access control system. These lists are used in some cases as explained in the following items:

When IZCloud Integration is enabled, an additional tab (Access Lists) will appear, depending on the RoadView version you are using (see Recommended Reading/Related Documents).

  • URL - The URL of the IZCloud account

  • Tenant ID - ID of the relevant IZCloud account

  • User Name - IZCloud account user name

  • Password - IZCloud account password

  • Ambiguity Chars - If an exact match to a plate cannot be done, Ambiguity (also called Normalization) is enabled. The characters in a plate read that could be mistaken for other characters will be changed to their associated ambiguity value during the plate-to-list matching process.
    Note that the list is bi-directional. For example, in addition to "8" being used instead of "B", "B" will be used instead of "8".

  • Push Video to Server - When enabled, RoadView sends images to the IZCloud server. You will also see an additional parameter:

    • Send Camera Image Timeout - How often images are sent to the IZCloud server.

  • Min Width for Fuzzy Match - If the number of characters in the LPR plate read is less than this value, then fuzzy matching will not be used to match this plate to the Vehicle List. Only exact or Ambiguity matching will be used.

  • Match On

    • Exact - An attempt is made to make an exact match, and an Ambiguity match - until all characters in the plate read match exactly to an item in the Vehicle List.

    • N-1 - An attempt is made to make an exact match, and an Ambiguity match - until all characters in the plate read match to an item in the Vehicle List – except for one wrong character.

    • N-2 - An attempt is made to make an exact match, and an Ambiguity match - until all characters in the plate read match to an item in the Vehicle List – except for two wrong characters.

  • Keep-Alive Enabled - When enabled, RoadView will sense the Keep Alive messages sent from the IZCloud server. If not enabled, these messages will be ignored.

    • Keep-Alive Interval - If RoadView does not receive a Keep Alive message from the IZCloud server for this amount of time, multiplied by the Keep-Alive Fail Count, then RoadView will re-establish the connection with the IZCloud server.

    • Keep-Alive Fail Count - The number of Keep-Alive Intervals for which RoadView will wait (if no Keep Alive messages are received from the IZCloud server) before re-establishing the connection with the IZCloud server.

  • Get Access List Timeout - How long RoadView waits after sending a request for access lists from the IZCloud server. If RoadView does not receive access lists for this amount of time, RoadView will stop sending the request.

  • Lanes

    • Lane ID - This table is automatically populated with the number of lanes defined in the Lanes section at the bottom of the Settings tab.

    • GPIO - The nn number that identifies the GPIOnn pin on the NVIDIA Jetson processor’s I/O module. A signal is sent to this pin when a RoadView plate read has a match with an IZCloud Vehicle List (expected plate reads). This is typically used to open a gate. See the NVIDIA Jetson documentation for further information.

    • EXIT - When enabled, all vehicles are allowed to pass through the device connected to the GPIO. This option is typically used for exit gates.

    • Remote URL - A request is sent to this URL when there is a match with an IZCloud Vehicle List (expected plate reads). This is typically used to open a gate via a web relay. The request is sent in a special format (see the Web Relay documentation referenced in Recommended Reading/Related Documents).

      In addition to the GPIO and Remote URL signals/requests, a signal is also sent to the camera; this signal can be used to open a gate via a relay (depends on the capabilities of the camera model).

  • Wiegand Controller (Converter) Enabled - For each plate in a Vehicle List, a Wiegand code can be added to use if there is an LPR match. (IZCloud also stores a Sentinel (No Match) Wiegand code if no plate match is found for any plate.)

    If there is a plate match, the Wiegand code will be used. Some cameras have the capability to send this code directly to an access control panel or device.

    However, if the access control panel is too far from the camera to be connected physically, or if the camera does not have Wiegand capabilities, then this option should be enabled. The Wiegand code will be sent via network to a Wiegand converter, which will then send the code to the access control panel or device.

    • Lane ID - This table is automatically populated with the number of lanes defined in the Lanes section at the bottom of the Settings tab.

    • WIC IP - The IP address of the Wiegand converter assigned to the Lane.

    • WIC Port - The port on the Wiegand converter used by RoadView to communicate with the Wiegand converter; 1001 is the default port on the Wiegand converter.
